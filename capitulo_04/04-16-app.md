## `src/App.tsx`

O arquivo `src/App.tsx` define o componente principal da aplicação. Em projetos React, o `App` costuma atuar como o primeiro componente de composição da interface, isto é, o ponto a partir do qual as páginas e estruturas principais passam a ser renderizadas.

No projeto atual, o `App.tsx` foi mantido com responsabilidade reduzida. Ele não controla estado, não realiza chamadas à API, não manipula formulários e não decide regras de cadastro, edição ou exclusão. Sua função é apenas montar a estrutura principal da aplicação e renderizar a página de clientes.

A importação de `ClientesPage` indica qual tela será exibida dentro da aplicação.

```tsx
import { ClientesPage } from "./pages/ClientesPage/ClientesPage";
```

Esse componente representa a página principal do cadastro de clientes. Ao ser importado no `App.tsx`, ele passa a compor a estrutura central da aplicação.

O arquivo também importa o CSS específico do `App`.

```tsx
import "./App.css";
```

Esse arquivo deve conter apenas estilos relacionados ao contêiner principal da aplicação. A separação evita que o `App.css` concentre regras pertencentes a componentes ou páginas específicas.

A função `App` retorna a estrutura principal da interface.

```tsx
export default function App() {
  return (
    <main className="app">
      <ClientesPage />
    </main>
  );
}
```

O elemento `main` representa semanticamente o conteúdo principal da aplicação. A classe `app` permite aplicar regras gerais de layout, como altura mínima ou organização do contêiner externo.

Dentro de `main`, o componente `ClientesPage` é renderizado. Isso significa que a página de clientes passa a ser o conteúdo principal da aplicação. O `App` não conhece os detalhes internos dessa página, não sabe como os clientes são carregados e não controla quais modais serão exibidos. Essas responsabilidades permanecem concentradas em `ClientesPage` e nos hooks utilizados por ela.

Assim, `App.tsx` atua como componente de composição inicial. Sua função é delimitar a área principal da aplicação e inserir a página responsável pelo cadastro de clientes. Essa redução de responsabilidade torna o arquivo simples, estável e coerente com a organização modular adotada no projeto.
